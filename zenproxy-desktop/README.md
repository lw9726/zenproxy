# ZenProxy Desktop 使用说明

这是 ZenProxy 的图形客户端。

压缩包里已经自带：

- `zenproxy-desktop.exe`：图形界面
- `zenproxy-client.exe`：本地魔改版 sing-box

解压后直接运行 `zenproxy-desktop.exe` 即可，不需要另外安装本地客户端。

## 这个客户端能做什么

- 启动和停止本地 `zenproxy-client`
- 从在线代理池预览节点
- 把选中的在线节点导入本地代理池
- 在本地代理池里批量绑定端口
- 管理本地订阅
- 支持明亮 / 暗黑主题

## 首次使用

1. 解压客户端压缩包
2. 双击运行 `zenproxy-desktop.exe`
3. 打开左侧“设置”
4. 填写下面几个项目：

- `服务器地址`
  - 例如：`https://zenproxy.top`
- `API Key`
  - 你的 ZenProxy 用户 API Key
- `本地控制地址`
  - 默认：`http://127.0.0.1:9090`
- `本地 API Secret`
  - 自己设置一个即可，默认不用改也行
- `zenproxy-client 路径`
  - 如果压缩包没有被拆散，通常会自动识别
- `端口起始` / `端口结束`
  - 本地绑定端口范围
  - 默认是 `20001-30000`
  - 如果你本机已经占用了 `200xx` 端口，可以改成例如 `60001-65535`

5. 点击“保存设置”
6. 回到“总览”页，点击“启动”

如果右上角显示：

- `进程运行中`
- `API 已连接`

说明本地客户端已经正常启动。

## 常用流程

### 1. 从在线代理池导入本地代理池

进入“在线代理池”：

1. 填获取数量、国家、类型等筛选条件
2. 点“预览”
3. 勾选你要的节点
4. 点“导入选中”

如果你想导入后立刻生成本地可用端口，点：

- `导入并自动绑定`

### 2. 使用本地代理池

进入“本地代理池”：

1. 查看已经导入的本地节点
2. 勾选需要使用的节点
3. 点“绑定选中”或“绑定全部”

绑定完成后，下方“当前绑定”里会显示端口，比如：

- `127.0.0.1:20001`
- `127.0.0.1:20002`

这些端口就是你程序里可以直接使用的本地代理端口。

每个端口同时支持：

- HTTP
- SOCKS5

### 3. 添加本地订阅

进入“本地订阅”：

1. 填订阅名称
2. 填订阅 URL
3. 或者直接粘贴订阅内容
4. 点“添加订阅”

导入成功后，订阅里的节点会自动进入“本地代理池”。

## 程序如何使用这些本地端口

例如你绑定出了一个端口 `20001`，那你的程序可以这样使用：

- HTTP 代理：`http://127.0.0.1:20001`
- SOCKS5 代理：`socks5://127.0.0.1:20001`

## 常见问题

### 启动失败

先检查：

- `zenproxy-client.exe` 路径是否正确
- 端口 `9090` 是否被别的程序占用
- 你设置的端口范围是否和本机已有服务冲突

### 能启动，但导入失败

先检查：

- `服务器地址` 是否正确
- `API Key` 是否正确
- 服务器是否可访问

### 绑定失败

一般是这几个原因：

- 端口范围太小
- 端口已被占用
- 本地节点数量太多，可用端口不够

这时把“端口起始 / 端口结束”改大一点，再重启本地客户端即可。
